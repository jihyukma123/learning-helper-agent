# 큰 목표

> "복잡한 비즈니스 문제를 본질적으로 꿰뚫어 보고, 가장 효율적이고 효과적인 소프트웨어로 제대로 해결하는 해결사가 된다."

# 고려해야되는 상황적 변수들

> AI AI AI

AI가 SDLC 전반에 걸쳐서 활용되면서 소프트웨어 개발 방식이 엄청 크게 변화하는 현재 시점에서 큰 목표에 부합하는 소프트웨어 엔지니어로 성장하는 방향성을 어떻게 잡아야하는지 고려 필요

# AI를 활용한 코딩이 보편화된 시대에 어떤 역량을 키워나가야 하는가?

지식과 경험 2가지 측면에서 역량 향상이 필요함

## 지식

인간이 담당해야하고, 프로덕트의 완성도와 사용성, 문제 해결력을 결정짓는 마지막 30%를 담당하기 위해서 필요한 지식들을 습득해야함

**기초적인 CS 지식**

- 자료구조와 알고리즘
- 분산 시스템 원칙
- 데이터베이스 내부 및 쿼리 최적화
- 네트워크 프로토콜과 보안

**환경에 대한 이해**

- 브라우저 환경
- 웹 아키텍처
- Cloud infrastructure

**모니터링 및 가시성**

- 배포되어 있는 앱의 사용 상태 등을 모니터링 하는 방법에 대한 이해

**테스팅**

- TDD
- AI-assisted coding 에서의 TDD 적용 경험

**소프트웨어 원칙 & best practices**

- 유지보수성
- 가독성
- 디자인 패턴

## 경험

**본질적 복잡성과, 우발적 복잡성을 분리해서 사고하고 해결하는 경험**

- 우발적 복잡성을 AI를 활용한다는 것의 의미 내재화. -> AI를 활용해서 코드를 생성할 때, 무분별하게 생성하는 것이 아니라, 일반적인 프로그래밍 원칙들(책임 분리, 모듈화 등)이 잘 지켜지고 있는지, 너무 과하게 어렵게 코드를 작성하지 않았는지 등등에 대해서 고민하고, 개선해서 적용하는 경험 필요
- 본질적 복잡성을 파악하려는 노력 필요 -> 회사에서도 얼마든지 할 수 있는 일. 주어진 일의 본질적 복잡성이 무엇인지 추출하는 연습

**장인 정신**

- 나만의 '장인 정신'이 무엇인지 고민하고, 내 이름으로 출력되는 모든 결과물에 그런 정신이 담기도록 노력

**프로그램을 많이 작성해보는 경험**

내가 사용하는 프로그램을 직접 작성

이를 통해서 코딩 경험 자체를 늘리기

**프로토타입이 아닌, 프로덕트를 만들어보는 경험**

진짜 서비스되는 프로덕트를 만든다고 생각하고, A~Z까지 만들면서 구성요소들이 어떻게 상호작용하는지, 기저에 어떤 기술들이 활용되는지 등에 대한 이해를 길러야함.

기능 구현 시 아래 사항들을 고려하는 능력 함양 필요

- 엣지케이스
- 모호한 부분
- 실패지점 예상 및 관리(네트워크 장애, )

**AI Detox**

주기적으로 AI 없는 개발 경험(너무 AI만 사용하다보면 기본적인 기술이 녹슬 가능성이 큼)

**커뮤니케이션 능력 향상 시도**

- 요구사항 수집 및 분석
  - 회사: 주어진 개발 작업의 완성도를 위해서 관계자들과 적극적으로 소통하는 경험
  - 개인: 프로덕트를 만들어보는 경험(작게라도, 사용자의 요구사항을 수집 - 분석하는 경험)
- 기술 문서 작성 및 문서화
  - 새로운 기술을 접했을 떄 이를 문서화하고 공유하는 활동을 지속적으로 해보자.
- 프로젝트 계획 및 추정
- 팀 리더십 및 멘토링..
