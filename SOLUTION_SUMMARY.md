### 문제 분석 (from issue)

**원인 분석**:

**미해결 질문**:

예시 (오늘 깨달은 "문제 정의가 먼저다"의 경우):
- A: 회사에서 기능 요청 받을 때마다 "왜 이게 필요하죠?"를 자동으로 물어보는 것?
- B: 뭔가 만들고 싶을 때 코드부터 치지 않고 문제부터 정의하는 습관?
- C: 일상의 모든 의사결정에서 "이게 진짜 문제인가?"를 생각하는 사고방식?

#### ✅ TODO 1: "체화"의 구체적 정의 내리기

**해결 방법**: `embodiment-framework.md` 문서 작성

**정의된 내용**:
1. **체화의 정의**: 깨달음이나 인사이트가 **의식적 노력 없이 자동으로 실행되는 행동 패턴**으로 전환된 상태

2. **체화의 3단계**:
   - 1단계: 인지 (Awareness) - "아, 이게 중요하구나"
   - 2단계: 의식적 적용 (Conscious Application) - "이 상황에서 이걸 써야지" (의도적 노력 필요)
   - 3단계: 체화 (Embodiment) - "자동으로 그렇게 행동함" (무의식적 실행)

3. **체화의 3가지 차원** (이슈의 A, B, C 질문에 대한 답):
   - **A. 반응적 체화 (Reactive Embodiment)**: 
     - 예: 회사에서 기능 요청 받을 때마다 "왜 이게 필요하죠?"를 자동으로 묻는 것
     - 특정 상황/트리거가 발생하면 자동으로 특정 행동이 나오는 것
     - If-Then 구조, 가장 먼저 체화되는 단계
   
   - **B. 습관적 체화 (Habitual Embodiment)**:
     - 예: 뭔가 만들고 싶을 때 코드부터 치지 않고 문제부터 정의하는 습관
     - 일상적으로 반복하는 활동에서 자연스럽게 패턴이 적용되는 것
     - 프로세스로 정착된 상태
   
   - **C. 사고방식 체화 (Mindset Embodiment)**:
     - 예: 일상의 모든 의사결정에서 "이게 진짜 문제인가?"를 생각하는 사고방식
     - 모든 의사결정의 기저에 깔린 사고 프레임으로 작동하는 것
     - 가장 깊은 수준의 체화, 세계관/철학 수준

4. **체화 성공 지표**:
   - 반응적: 트리거 발생 시 90% 이상 자동 실행
   - 습관적: 해당 활동 시작 시 자연스럽게 적용
   - 사고방식: 모든 관련 상황에서 자동으로 해당 렌즈로 사고

**파일 위치**: `.opencode/skill/action-trigger/references/embodiment-framework.md`

#### ✅ TODO 2: 깨달음 → 행동 트리거로 전환하는 시스템 설계

**해결 방법**: `action-trigger` 스킬 및 구조화된 프레임워크 개발

**시스템 구성 요소**:

1. **깨달음 재구성 (Insight Clarification)**:
   - 핵심 원칙을 한 문장으로 정리
   - 왜 이게 중요한지
   - 이게 해결하려는 문제가 무엇인지

2. **체화 차원 분석 (Embodiment Dimension Analysis)**:
   - 3가지 차원(반응적/습관적/사고방식)에서 분석
   - 각 깨달음에 적합한 차원 선택

3. **구체적 행동 트리거 설계 (Action Trigger Design)**:
   ```
   📍 상황 (Situation)
     - 언제: [시점]
     - 어디서: [장소/환경]
     - 무엇이 트리거가 되는가: [구체적 신호]

   ⚡ 행동 (Action)
     - 정확히 무엇을 할 것인가: [관찰 가능한 행동]
     - 소요 시간: [예상 시간]
     - 준비물/도구: [필요한 것]

   📊 측정 (Measurement)
     - 어떻게 확인할 것인가: [추적 방법]
     - 성공 기준: [목표 수치]
     - 추적 주기: [일일/주간/월간]
   ```

4. **리마인더 시스템**:
   - 환경 단서 (물리적 공간)
   - 디지털 리마인더 (앱, 알림)
   - 사람 활용 (동료/친구)
   - 프로세스 통합 (기존 루틴에 끼워넣기)

**시스템의 핵심 원칙**:
- **트리거 구체화**: "좋은 생각이네" → "언제, 어디서, 무엇을 할 것인가"로 변환
- **측정 가능성**: 체화 여부를 객관적으로 확인 가능
- **점진적 확장**: 반응적 → 습관적 → 사고방식 순으로 확장

**파일 위치**: `.opencode/skill/action-trigger/SKILL.md`

#### ✅ TODO 3: 반복/습관화를 위한 방법 탐색

**해결 방법**: 체화 로드맵 및 추적 시스템 설계

**체화 로드맵** (3단계 프로세스):

1. **Phase 1: 의식적 적용 (1-2주차)**:
   - 목표: 트리거 인식 & 의도적 행동 실행
   - 성공 기준: 트리거 발생 시 30% 행동 실행
   - 도구: 리마인더, 일일 체크리스트

2. **Phase 2: 반복 강화 (3-4주차)**:
   - 목표: 실행률 50% 이상
   - 성공 기준: 의식적으로는 항상 기억하고 시도함
   - 도구: 주간 회고, 패턴 분석

3. **Phase 3: 자동화 (5주차 이상)**:
   - 목표: 의식적 노력 없이 자동 실행
   - 성공 기준: 실행률 90% 이상
   - 도구: 정기 점검, 행동 패턴 관찰

**추적 템플릿**:
- 주간 기록 양식 제공
- 실행률 계산
- 주간 회고 프레임워크
- 월간 체화 진행 상황 점검

**습관화 지원 메커니즘**:
1. **환경 설계**: 물리적/디지털 환경에 리마인더 배치
2. **측정 시스템**: 실행 여부를 추적하고 피드백 제공
3. **점진적 목표**: 30% → 60% → 90% 단계적 목표 설정
4. **현실적 기대**: 완벽주의 지양, 점진적 개선 강조

## 사용 방법

AI Agent에게 다음과 같이 요청:

```
"오늘 '문제 정의가 먼저다'는 걸 깨달았는데, 
이걸 실제로 어떻게 적용할 수 있을까?"
```

Agent는 `action-trigger` 스킬을 활성화하여:
1. 깨달음 재구성
2. 체화 차원 분석
3. 구체적 행동 트리거 설계
4. 체화 로드맵 제공
5. 리마인더 시스템 제안
6. 추적 템플릿 제공
